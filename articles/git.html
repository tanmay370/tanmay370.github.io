Git Intro Commands

29 August 2023 by Tanmay

Initializing a project with git

  • git status

    On branch master --> Branch named master
    nothing to commit , working tree clean

  • git init

    init command initialise a .git repository .All files & folders fall under untracked section.

  • Add File

    File is added to the staging area & files and folders fall under unmodified section. Shortcut way to add all available files to the staging area is to run command 

     git add .

  • Commit (Taking screenshot)

    Commiting (taking screenshot) of code exist that time when we run the command below here -m flag stands for message and "Initial commit" stands for comment user defined. 

     git commit -m "Initial commit"

  • Delete or Unmake a repo

    Delete a file or folder
    .git is like a container contained compressed snapshot of previous version 

     git rm filename
    From working tree and staging area
     git rm --cached filename

Branches in Git

29 August 2023 by Tanmay

  • To check how many branches are there
    Run command 
     git branch
  • To make a branch[feature] apart from master branch
    Run command here "feature" is user defined 
     git branch feature
  • To go to a particular branch (for example feature) 
     git checkout feature
  • To merge branch with master branch [Here BranchName --> feature] 
     git merge BranchName
  • To make a branch and go into branch directly 
     git checkout -b BranchName

How to Connect Remote repo[Github] to master/local repo [local computer]

30 August 2023 by Tanmay

  • How to connect Remote repo and master [local repo]. Here origin is user-defined name given to this URL
    Run command 
     $ git remote add origin URL
  • How to check remote repository
    Run command 
     git remote
  • To know the URL of Push & Fetch
    Run command 
     git remote -v
  • To change (set) URL
    Run command 
     $ git remote set-url origin URL
  • To push (or add) our branches with remote (origin) 
     $ git push -u origin branchName
  • To push (or connect) our master branch with remote repo (origin)
    Run command 
     git push -u origin master

    error: failed to push some refs to 'https://github.com/YourRepoName/GitHubUsername.github.io.git'

    Occurs because origin(URL) is private repo. We want to have read-write head over origin [private repo].

    • Steps to get Read-Write head
  • Generating SSH Key Generating a new SSH Key Then follow the steps : 
    1.  ssh-keygen -t ed25519 -C "your_email@example.com"
    2.  > Enter a file in which to save the key (/c/Users/YOU/.ssh/id_ALGORITHM):[Press enter]
    3.  Enter passphrase (empty for no passphrase): 
 > Enter same passphrase again: 
    4.  eval "$(ssh-agent -s)"
    5.  ssh-add ~/.ssh/id_rsa
    6.  cat ~/.ssh/id_rsa.pub  
 or
 ssh-add ~/.ssh/id_ed25519

    After these steps in Git Bash terminal a long key [text] starting from ssh-ed25519 AAA...yourEmaiid appears. Copy that key and paste it under Key label

    Then Run command 

     git push -u origin master
